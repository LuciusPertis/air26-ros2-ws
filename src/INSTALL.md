# AIR26 Workshop — Full Install & Build Guide

Everything you need to go from a **fresh Ubuntu 22.04 + ROS 2 Humble** box to a
workspace where **all projects (01–07) compile in one shot**, plus the extra
(non-ROS) runtimes each demo needs at run time.

> **TL;DR** — clone, then:
> ```bash
> cd ~/air26-ros2-ws
> ./src/build_all.sh              # rosdep + colcon build EVERYTHING
> source install/setup.bash
> ```
> That gets every ROS package to compile. The **simulators / LLM / VLA / firmware**
> are separate runtimes — see [§3](#3-per-project-runtime-extras-not-built-by-colcon).

---

## 0. Prerequisites

| Requirement | Notes |
|-------------|-------|
| Ubuntu 22.04 (Jammy) | ROS 2 Humble's target |
| ROS 2 **Humble** | `source /opt/ros/humble/setup.bash` — [install guide](https://docs.ros.org/en/humble/Installation.html) (`ros-humble-desktop` recommended) |
| `colcon` + `rosdep` | `sudo apt install python3-colcon-common-extensions python3-rosdep` |
| `git`, `python3-pip` | `sudo apt install git python3-pip` |
| Disk | ~15 GB free (ROS build + MuJoCo + Ollama models + SmolVLA weights + Webots) |

```bash
# one-time ROS base (skip anything you already have)
sudo apt update
sudo apt install -y ros-humble-desktop \
    python3-colcon-common-extensions python3-rosdep \
    git python3-pip
```

## 1. Get the workspace

```bash
git clone <repo-url> ~/air26-ros2-ws
cd ~/air26-ros2-ws
```

The Stretch upstream stack (`src/04_stretch-hr-se3/upstream/`) is **vendored**
(committed at pinned commits), so there is nothing extra to clone. Build
artifacts (`build/ install/ log/`) and PlatformIO caches (`**/.pio/`) are
gitignored and regenerated by the build.

---

## 2. Build everything in one shot (ROS side)

The convenience script resolves all ROS dependencies with `rosdep`, then
`colcon build`s the whole workspace:

```bash
./src/build_all.sh
```

What it does (and you can run these by hand):

```bash
source /opt/ros/humble/setup.bash

# rosdep pulls all the ros-humble-* apt packages the manifests declare:
# webots_ros2, cv_bridge, vision_msgs, py_trees(_ros), nav2_*, slam_toolbox,
# moveit_*, ros_gz_*, control_msgs, xacro, rviz2, robot_state_publisher, …
sudo rosdep init 2>/dev/null || true
rosdep update
rosdep install --from-paths src --ignore-src -r -y

# build all packages (01–07); COLCON_IGNORE'd dirs (firmware, mini-VLA) skip
colcon build --symlink-install
source install/setup.bash
```

**Script flags**

| Flag | Effect |
|------|--------|
| _(none)_ | rosdep + build all |
| `--no-rosdep` | skip rosdep (deps already installed) |
| `--pip-extras` | also `pip`-install the easy sim runtimes (MuJoCo pins + Stretch sim + Ollama client) — see §3 |
| `--packages "a b c"` | build only these packages |
| `-h` | help |

> **Heavy packages:** project **04** (MoveIt 2 + Nav 2) and the vendored
> `stretch_ros2` are the long poles of the build. If you only want the light
> projects first: `./src/build_all.sh --packages "basics_py basics_cpp basics_cross"`.

After this step, `ros2 pkg list` shows all workshop packages and every project
**compiles**. The remaining sections add the runtimes needed to actually **run**
each sim/demo.

## 3. Per-project runtime extras (NOT built by colcon)

These are simulator engines, model servers, virtualenvs and MCU toolchains —
not ROS packages, so `colcon`/`rosdep` don't install them. Each project's own
`SETUP.md` is the authoritative, verified record; this is the consolidated list.

### 3.1 MuJoCo (projects 02, 04, 05, 07)

Python MuJoCo, pinned to protect ROS Humble's `numpy 1.24`:

```bash
python3 -m pip install \
  "numpy==1.24.2" "opencv-python==4.10.0.84" "mujoco==3.2.6" "ros2-numpy==0.0.5"
```
> `ros2-numpy` comes from **PyPI** (there is no `ros-humble-ros2-numpy` apt pkg).
> `mujoco==3.2.6` is the upstream Stretch pin; it runs the newer model fine.
> (`./src/build_all.sh --pip-extras` runs exactly this.)

### 3.2 Stretch MuJoCo sim (project 04)

`stretch_mujoco` (vendored under `upstream/`) is an **editable** install whose
build backend is `uv`:

```bash
python3 -m pip install uv
python3 -m uv pip install --system -e \
  ~/air26-ros2-ws/src/04_stretch-hr-se3/upstream/stretch_mujoco
```
> Plain `pip install -e` **cannot** drive `uv_build` in its build isolation —
> use `uv` as above. Full detail + pinned commits: `04_stretch-hr-se3/SETUP.md`.

### 3.3 Webots R2025a (project 05 — primary sim)

```bash
sudo apt-get install -y ros-humble-webots-ros2 ros-humble-cv-bridge \
     ros-humble-vision-msgs ros-humble-image-transport \
     ros-humble-compressed-image-transport
wget https://github.com/cyberbotics/webots/releases/download/R2025a/webots_2025a_amd64.deb
sudo apt-get install -y ./webots_2025a_amd64.deb     # ~2 GB → /usr/local/webots
```
> `webots_ros2` **2025.0.0** must pair with **Webots R2025a** (versions match).
> Headless? render camera under software GL with
> `xvfb-run -a -s "-screen 0 1280x1024x24" ros2 launch perceptbot_sim webots.launch.py`.
> See `05_perception/SETUP.md`.

### 3.4 Gazebo Fortress (projects 02, 05 — reference sim)

The `ros_gz_bridge` / `ros_gz_sim` packages are pulled by rosdep; install the
Ignition/Gazebo Fortress engine itself per the
[official instructions](https://gazebosim.org/docs/fortress/install_ubuntu).
(Gazebo paths in 02/05 are reference/deferred — MuJoCo + Webots are the primary
demos.)

### 3.5 Ollama LLM (project 06)

```bash
curl -fsSL https://ollama.com/install.sh | sh   # installs + enables systemd service
ollama pull qwen3:1.7b                           # ~1.4 GB (default, fast on CPU)
ollama pull qwen3:4b                             # ~2.5 GB (optional, more accurate/slower)
pip install ollama                               # python client used by the package
```
> The server runs as a **systemd service** on `http://localhost:11434` (no need
> to run `ollama serve`). Offline? the package has a `backend:=mock` keyword path
> that needs no model/GPU. See `06_llm-integration/SETUP.md`.

### 3.6 SmolVLA venv (project 07)

`torch`/`lerobot` want `numpy>=2`, which conflicts with ROS's `numpy 1.24`, so
they live in a **dedicated venv** (`/home/lsp/vla_venv`); the launch file runs
the policy node with that interpreter.

```bash
python3 -m venv --system-site-packages /home/lsp/vla_venv
source /home/lsp/vla_venv/bin/activate
pip install --upgrade pip wheel setuptools
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu   # no GPU
pip install "lerobot[smolvla]"
pip install "numpy<2"      # re-pin for rclpy (harmless opencv/rerun "want >=2" warnings)
deactivate
# smoke test:
/home/lsp/vla_venv/bin/python -c \
  "from lerobot.policies.smolvla.modeling_smolvla import SmolVLAPolicy; print('ok')"
```
> First policy load downloads `smolvla_base` + `SmolVLM2-500M` (~2 GB). CPU
> inference is ~25 s/chunk. See `07_vla_demo/SETUP.md`.

### 3.7 PlatformIO firmware — ESP32 (projects 02 & 05)

The ESP32 / ESP32-CAM firmware under `src/<project>/firmware/` is **outside
colcon** (`COLCON_IGNORE`) and built with **PlatformIO**. Use the **official**
installer — a bare `pip install platformio` does **not** create
`~/.platformio/penv` and breaks the `micro_ros_platformio` build.

```bash
sudo apt-get install -y python3.10-venv dbus-x11
# official installer (creates ~/.platformio/penv):
curl -fsSL -o /tmp/get-platformio.py https://raw.githubusercontent.com/platformio/platformio-core-installer/master/get-platformio.py
python3 /tmp/get-platformio.py
# build + flash (from a firmware dir, e.g. src/02_micro_ros/firmware/esp32_microbot):
~/.platformio/penv/bin/pio run -t upload      # compile + flash over USB serial
```
> micro-ROS then talks to the ROS host over **WiFi/UDP**. Per-board detail:
> `src/02_micro_ros/firmware/*/README.md` and `src/05_perception/firmware/*/`.

## 5. Dependency map (what installs what)

| Layer | Tool | Covers |
|-------|------|--------|
| ROS packages | **rosdep** (`build_all.sh`) | all `ros-humble-*` deps: webots_ros2, cv_bridge, vision_msgs, py_trees(_ros), nav2_*, slam_toolbox, moveit_*, ros_gz_*, control_msgs, xacro, rviz2 … |
| Sim (Python) | **pip** (`--pip-extras`) | MuJoCo, opencv, ros2-numpy, Stretch `stretch_mujoco` (via `uv`) |
| Sim (engine) | **apt / .deb** | Webots R2025a, Gazebo Fortress |
| LLM | **Ollama** | server (systemd) + qwen3 models + python client |
| VLA | **venv** (`/home/lsp/vla_venv`) | torch, lerobot[smolvla] |
| Firmware | **PlatformIO** (official installer) | ESP32 / ESP32-CAM micro-ROS (outside colcon) |

Each project's `SETUP.md` (and `firmware/*/README.md`) holds the verified,
machine-specific detail; this file is the top-level index students start from.
